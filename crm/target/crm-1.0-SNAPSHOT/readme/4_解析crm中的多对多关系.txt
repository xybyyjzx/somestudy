我们现在举例
人和职业
一个人可以同时拥有多份职业，一份职业可同时让多个人去做

tbl_person
id              name
A0001           zs
A0002           ls
A0003           ww

tbl_job
id              name
B0001           工人
B0002           农民
B0003           教师

zs是工人，也是农民
ls是工人，也是教师
ww是工人，也是农民，也是教师

多对多的关系，需要我们建立第三张表，关联关系表
这张关联关系表，没有实际字段相关的意义，仅仅只是为了维护表与表之间的关系
关联关系表，由3个字段所组成，分别是表主键id，以及另外需要维护的两张表的外键



tbl_person_job_relation 人和职业的关联关系表
